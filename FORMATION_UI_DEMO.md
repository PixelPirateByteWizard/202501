# 阵型界面UI优化演示

## 界面设计特点

### 1. 背景设计
- 使用 `assets/bg/BG_7.png` 作为背景图片
- 营造沙漠战场的氛围感

### 2. 顶部资源栏
- 仿照游戏界面设计，显示玩家信息和资源
- 左右两侧的群组按钮
- 中间显示玩家名称和金币数量
- 采用棕色渐变背景和金色边框

### 3. 阵型布局区域
- 3x3网格布局，展示9个阵型位置
- 每个位置显示为圆形底座
- 已部署武将显示角色图片和等级标识
- 空位置显示添加图标和位置名称

### 4. 底部武将选择面板
- 横向滚动显示可用武将
- 武将卡片显示角色图片、等级和名称
- 支持点击选择武将
- 选中状态有视觉反馈

### 5. 操作按钮区域
- 7个圆形按钮：返回、武将、竞技、商店、成就、群组、设置
- 采用棕色渐变和金色边框设计

- 现在使用Role_1.png到Role_32.png的新图片文件
- 包含13个有数据的角色（赵云、吕布、司马懿等）

## 技术实现

### 布局优化
- 修复了顶部资源栏的溢出问题
- 使用Expanded包装右侧文本区域，防止布局溢出
- 调整了间距和字体大小，确保在不同屏幕尺寸下正常显示
- 添加了文本溢出处理（ellipsis）

### 资源配置
```yaml
assets:
  - assets/bg/
  - assets/role/
```

### 主要组件
- `_buildTopResourceBar()`: 顶部资源栏
- `_buildFormationArea()`: 阵型布局区域
- `_buildFormationGrid()`: 3x3阵型网格
- `_buildPositionSlot()`: 单个阵型位置
- `_buildBottomPanel()`: 底部武将选择面板
- `_buildActionButton()`: 操作按钮

### 状态管理
- `_selectedGeneralIndex`: 当前选中的武将索引
- `_currentFormation`: 当前阵型配置
- `_availableGenerals`: 可用武将列表

## 
问题修复

### 布局溢出修复

#### 第一次修复（29像素溢出）
- **问题**：顶部资源栏Row布局溢出29像素
- **原因**：右侧"机敏招佳人"文本过长，固定宽度布局无法适应
- **解决方案**：
  1. 将右侧资源区域包装在Expanded中，使其自适应剩余空间
  2. 减小字体大小从16px到14px
  3. 添加maxLines和overflow处理，防止文本溢出
  4. 调整间距从16px到12px，节省空间
  5. 缩小中间装饰容器尺寸

#### 第二次修复（17像素溢出）
- **问题**：左侧金币数字"254230031"过长导致17像素溢出
- **原因**：长数字在固定宽度Row中无法完全显示
- **解决方案**：
  1. 对左侧金币数字使用Expanded包装
  2. 对右侧金币数字使用Flexible包装
  3. 添加数字格式化函数，将大数字转换为K/M格式（如254.2M）
  4. 减少容器padding和间距
  5. 为所有文本添加溢出处理

### 响应式设计
- 使用Expanded和Flexible确保在不同屏幕尺寸下正常显示
- 文本溢出时显示省略号
- 灵活的间距和尺寸调整
- 大数字自动格式化为简洁形式（K/M后缀）

### 数字格式化
- 添加了`_formatNumber()`函数
- 1000以上显示为K格式（如1.5K）
- 1000000以上显示为M格式（如254.2M）
- 节省显示空间，提高可读性

现在界面在各种屏幕尺寸下都能正常显示，不会出现布局溢出问题，大数字也能以简洁的方式展示。
