# 武将升级系统使用指南

## 功能概述

新的武将升级系统引入了材料需求机制，玩家需要消耗特定材料和银币才能升级武将。

## 主要特性

### 1. 材料系统
- **经验卷轴**：提供经验值的消耗品
  - 小型经验卷轴：少量经验
  - 中型经验卷轴：中等经验
  - 大型经验卷轴：大量经验

- **升级石**：武将升级必需材料
  - 普通升级石：1-10级武将使用
  - 稀有升级石：11-20级武将使用
  - 史诗升级石：21级以上武将使用

- **银币**：通用货币，所有升级都需要消耗

### 2. 升级需求计算
升级所需材料根据武将等级和稀有度动态计算：

- **银币需求**：100 + (当前等级 × 50) + (稀有度 × 100)
- **升级石需求**：根据等级段选择对应类型
- **经验卷轴需求**：根据稀有度确定数量和类型

### 3. 升级流程

#### 步骤1：检查条件
- 武将经验值必须达到当前等级上限
- 拥有足够的升级材料
- 拥有足够的银币

#### 步骤2：升级确认
- 显示升级预览对话框
- 展示属性提升详情
- 列出所需材料和当前拥有量
- 确认后执行升级

#### 步骤3：升级执行
- 消耗相应材料和银币
- 提升武将等级和属性
- 播放升级成功音效
- 显示升级结果

  - 灰色：条件不满足

### 武将详情界面
- 详细属性展示
- 升级预览功能
- 材料需求列表
- 一键升级操作

- 银币余额展示
- 材料详情查看
- 数量统计

## 材料获取方式

### 初始材料
新玩家开始游戏时获得：
- 小型经验卷轴 × 10
- 普通升级石 × 5
- 银币 × 1000

### 战斗奖励
- 胜利后获得经验和银币
- 特殊战斗可能掉落升级材料

### 商店购买
- 使用银币购买各类材料
- 稀有材料价格较高

## 技术实现

### 核心类
- `Material`：材料基础类
- `MaterialStack`：材料堆叠（包含数量）
- `UpgradeRequirement`：升级需求
- `UpgradeResult`：升级结果

### 服务类
- `MaterialService`：材料管理服务
- `GeneralService`：武将服务（扩展升级功能）
- `GameDataService`：游戏数据服务（支持材料存储）

### 界面组件
- 响应式升级确认对话框
- 材料不足提示对话框
- 升级成功庆祝对话框
- 材料详情展示组件

## 使用示例

### 检查是否可升级
```dart
bool canUpgrade = GeneralService.canUpgrade(general, gameState);
```

### 执行升级
```dart
final result = GeneralService.upgradeGeneral(general, gameState);
if (result.success) {
  // 升级成功处理
} else {
  // 显示错误信息
  showDialog(context, result.message);
}
```

### 获取升级预览
```dart
final preview = GeneralService.getUpgradePreview(general);
print('新等级: ${preview['newLevel']}');
print('属性变化: ${preview['statChanges']}');
```

## 平衡性设计

### 材料稀缺性
- 普通材料：容易获得，满足基础升级需求
- 稀有材料：获取困难，限制高级武将升级速度
- 银币：通过战斗和任务获得，形成资源管理策略

### 升级成本递增
- 等级越高，所需材料越多
- 稀有度越高，升级成本越大
- 鼓励玩家合理分配资源

### 属性成长差异化
- 不同职业武将属性成长不同
- 前锋：武力成长最高
- 谋士：智力成长最高
- 辅助：统率成长最高
