4. 设计系统与UI/UX原则 (Design System & UI/UX Principles)

(色彩、字体、空间、动效等原则保持不变，新增战斗视觉原则)

新增 - 战斗视觉原则:

核心隐喻 (Core Metaphor): 沙盘推演 (Sand Table Simulation)。整个战斗界面是一个俯瞰视角的战略沙盘。所有图形元素都服务于“排兵布阵、运筹帷幄”的核心体验，而非“观看个人战斗动画”。

图形风格: 符号化与意象化。我们不画“人”，我们画“棋子”；我们不画“火焰”，我们画“燃烧的符号”。所有图形都应是高度概括、具有东方美学特征的符号，与我们的文字风格融为一体。

信息优先级: 态势 > 细节。图形的首要任务是清晰地传达战场布局、兵力对比、关键状态等宏观态势信息。具体的战斗细节、武将的呐喊、刀剑的碰撞，则交还给文字去描绘。

C. 核心任务流程拆解 (Core Task Flow Breakdown)

任务名称: 进行一场“官渡之战”

界面布局: 屏幕一分为二。

上半区 (70%): 沙盘区 (The Sand Table)。这是一个带有淡雅网格的俯瞰战场。

下半区 (30%): 战报与指令区 (Log & Command Area)。这里依然是我们引以为傲的AI实时文字直播和玩家的指令按钮。

沙盘区 - 视觉元素详述:

单位表现: 每个“武将”在沙盘上都表现为一个 stylized “棋子”。这个棋子可以设计成一个带有底座的圆形或方形木牌质感图标。棋子的正中，是一个苍劲有力的书法汉字，代表该武将的姓氏（如“关”、“张”、“曹”）。我方棋子为青玉色，敌方为赭石色。

信息传达:

兵力/血量: 每个棋子下方都有一条极简的横线，代表兵力条，颜色与棋子主色一致。兵力减少时，横线从右至左缩短。

状态效果 (Buff/Debuff): 棋子上方会浮现出极简的线性图标来表示状态。例如：一个“盾”形图标代表“防御提升”，一个“火焰”图标代表“燃烧中”，一个“酒杯”图标代表“醉酒/混乱”。所有图标风格与UI主图标系统保持统一。

当前行动单位: 轮到哪个棋子行动，它的底座会有一个柔和的光晕呼吸效果，引导玩家的视觉焦点。

战斗动画 (极简):

普通攻击: 攻击方棋子向目标棋子之间，会有一道迅捷的墨色细线一闪而过。

技能释放: 释放范围技能时（如“火计”），对应的沙盘格子上会出现一个朱砂色的阵法图案，短暂亮起后消失。

暴击/斩杀: 攻击的墨线会变为醒目的朱砂色，被攻击的棋子会有一个碎裂的视觉特效，随后从沙盘上移除。

战报与指令区:

AI文字直播: 此区域的功能保持不变，AI将继续生成文学化的战斗描述。关键在于，文字描述将与沙盘上的视觉表现完美同步。 例如，当沙盘上“关”字棋子向“颜良”棋子发动攻击时，下方的文字会同步出现：“电光火石间，关羽凤目圆睁，青龙偃月刀如一道惊鸿，直劈颜良顶上！”

指令按钮: 玩家的“攻击”、“计策”、“防御”、“道具”等指令按钮清晰地陈列在此区域。

魔鬼细节 (Micro-interaction): 当玩家长按沙盘上的任意一个敌方棋子时，沙盘上会用虚线标示出它的可攻击范围，同时下方的战报区会临时浮现出该单位的“核心信息卡”（如：【颜良 | 河北名将 | 特点：勇冠三军，有勇少谋】），帮助玩家快速决策。

步骤4：战斗结果与反馈 (战后总结页): (与V3.0一致，但现在玩家可以回顾带有图形的战斗棋局复盘)
